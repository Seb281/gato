## Build and Development Commands

```bash
pnpm dev          # Start development mode with hot reload
pnpm build        # Build for production
pnpm compile      # Type-check without emitting
```

After running `pnpm build`, load the `.output/chrome-mv3` folder as an unpacked extension in Chrome (`chrome://extensions` → Developer mode → Load unpacked).

## Architecture

This is a Chrome Manifest V3 extension for context-aware text translation, built with WXT, React, TypeScript, and Tailwind CSS v4.

### Entry Points (WXT Convention)

- **Background Service Worker** (`src/entrypoints/background/index.ts`): Handles message passing between popup/content scripts, translation API calls, and authentication
- **Content Script** (`src/entrypoints/content/index.tsx`): Injected into user-approved sites (via `registerContentScripts`) or one-shot via context menu. Manages text selection detection, tooltip display, and renders the translation popup
- **Popup** (`src/entrypoints/popup/`): Extension popup UI for per-site activation, settings (target language, personal context), and allowed sites management

### Message Flow

1. User selects text → content script detects via `mouseup` event → shows tooltip
2. User clicks tooltip or right-clicks and selects "Translate" → content script sends `translate` message to background
3. Background fetches from external translation API (configured via `VITE_BASE_URL`)
4. Response rendered in `TranslationPopup` component overlaying the page

### Activation Model

- **Per-site whitelist**: Users enable the extension on specific sites via the popup. Uses `optional_host_permissions` + `chrome.scripting.registerContentScripts()` for persistent injection that survives page refresh and browser restart.
- **Context menu (right-click)**: Works on any page without pre-activation. Uses `activeTab` permission granted by the context menu click for one-shot injection.
- **Allowed sites** stored in `chrome.storage.sync` under key `allowedSites` as an array of match patterns.

### Key Directories

- `src/entrypoints/background/helpers/`: Background script logic (translation, login, token management)
- `src/entrypoints/content/helpers/`: Content script utilities (text selection expansion, language detection, highlighting)
- `src/entrypoints/content/components/`: Content script React components (TranslationPopup, Tooltip)
- `src/components/ui/`: Shared UI components (shadcn/ui style with Radix primitives)
- `src/lib/utils.ts`: Utility functions (cn for className merging)

### Configuration

- **WXT Config**: `wxt.config.ts` - extension build configuration
- **Path alias**: `@/` maps to `./src/` (auto-configured by WXT)
- **Environment variable**: `VITE_BASE_URL` for translation API endpoint
- **User settings**: stored in `chrome.storage.sync` (`targetLanguage`, `sourceLanguage`, `personalContext`, `allowedSites`)
- **Auth tokens**: stored in `chrome.storage.local`

### Extension Manifest

Generated by WXT based on entrypoint definitions. Uses Manifest V3 with:

- Permissions: storage, tabs, activeTab, scripting, contextMenus
- Optional host permissions: `https://*/*`, `http://*/*` (requested per-site at runtime)

### Turborepo Integration

This extension is part of a monorepo managed by Turborepo:
- Run from root: `pnpm --filter context-aware-translator-extension dev`
- Or from this directory: `pnpm dev`
